Fondamentaux – DataFrame & Structures
DataFrame — Structure Bidimensionnelle

Un DataFrame est la structure centrale de Pandas. C'est un tableau 2D avec des lignes (index) et des colonnes nommees, similaire a une feuille Excel mais beaucoup plus puissant.

Caracteristiques

  • Colonnes pouvant avoir des types differents (int, float, object, datetime, bool)
  • Taille mutable : on peut ajouter/supprimer lignes et colonnes
  • Index etiquete pour les lignes, noms pour les colonnes
  • Operations vectorisees : pas besoin de boucles for
  • Gestion native des valeurs manquantes (NaN)
  • Supporte les jointures, pivot, groupby

Series vs DataFrame

  • Series : tableau 1D etiquete — une seule colonne
  • DataFrame : tableau 2D — plusieurs colonnes
  • Une colonne d'un DataFrame est une Series
  • df['col'] retourne une Series
  • df[['col']] retourne un DataFrame
import pandas as pd
import numpy as np

Selection & Indexation des Donnees
# Selection de colonnes
df['note']              # une colonne → Series
df[['nom', 'note']]   # plusieurs colonnes → DataFrame
df.note                 # attribut (fonctionne sans espace dans le nom)

# Selection de lignes
df[0:5]                 # slicing par position
df.iloc[2]             # ligne a la position 2
df.iloc[0:5, 1:3]      # lignes 0-4, colonnes 1-2
df.loc['a']            # ligne avec index 'a'
df.loc['a':'c', 'nom':'note']  # par etiquettes

# Selection de cellules specifiques
df.at['a', 'note']     # cellule par etiquettes (rapide)
df.iat[0, 1]            # cellule par position (rapide)

# Filtres booleens
df[df['note'] > 10]
df[(df['note'] > 10) & (df['note'] < 18)]   # ET
df[(df['note'] < 8) | (df['note'] > 18)]    # OU
df[~(df['note'] < 10)]                       # NOT
df[df['ville'].isin(['Casa', 'Rabat'])]      # IN
df.query('note > 10 and ville == "Casa"')     # query SQL-like
Difference .iloc vs .loc : .iloc = position numerique (0, 1, 2...). .loc = etiquette/label de l'index. Si l'index est numerique, ils peuvent donner le meme resultat, mais attention aux slices : iloc[0:3] = lignes 0,1,2 ; loc[0:3] = lignes 0,1,2,3 (inclus).

GroupBy — Agregation par Groupes

Le groupby divise le DataFrame en groupes selon une ou plusieurs colonnes, puis applique une fonction a chaque groupe (Split → Apply → Combine).

# Syntaxe de base
df.groupby('categorie')['ventes'].sum()
df.groupby('categorie').mean(numeric_only=True)

# Multiple colonnes de groupage
df.groupby(['region', 'produit']).agg({
    'ventes': ['sum', 'mean', 'count'],
    'prix': 'mean'
})

# agg avec fonctions nommees
df.groupby('cat').agg(
    total_ventes=('ventes', 'sum'),
    prix_moyen=('prix', 'mean'),
    nb_produits=('produit', 'count')
)

# Fonctions personnalisees
df.groupby('cat')['note'].apply(lambda x: x.max() - x.min())  # amplitude

# transform() : conserve la meme forme que le DataFrame original
df['mean_par_cat'] = df.groupby('cat')['ventes'].transform('mean')

# filter() : garde seulement les groupes satisfaisant une condition
df.groupby('cat').filter(lambda x: x['ventes'].sum() > 1000)
Fonctions d'agregation disponibles : sum(), mean(), count(), min(), max(), std(), var(), median(), first(), last(), nunique(), size()

Interpretation de la correlation :
0.7 a 1.0 — Forte correlation positive (les deux variables augmentent ensemble)
0.3 a 0.7 — Correlation moderee

Jointures & Fusion de DataFrames
# merge() — equivalent JOIN SQL
pd.merge(df1, df2, on='id', how='inner')   # INNER JOIN
pd.merge(df1, df2, on='id', how='left')    # LEFT JOIN
pd.merge(df1, df2, on='id', how='right')   # RIGHT JOIN
pd.merge(df1, df2, on='id', how='outer')   # FULL OUTER JOIN
pd.merge(df1, df2, left_on='id1', right_on='id2')  # colonnes differentes

# concat() — empilement vertical ou horizontal
pd.concat([df1, df2], axis=0)   # empiler les lignes
pd.concat([df1, df2], axis=1)   # empiler les colonnes
pd.concat([df1, df2], ignore_index=True)  # reinitialiser l'index
Rappel SQL : INNER = seulement les lignes communes | LEFT = toutes lignes de gauche + correspondances droite (NaN si absent) | OUTER = toutes les lignes des deux tables

# Strategies de traitement selon le contexte

# 1. Remplacement par la moyenne (variable continue)
df['age'] = df['age'].fillna(df['age'].mean())

# 2. Remplacement par la mediane (si outliers)
df['salaire'] = df['salaire'].fillna(df['salaire'].median())

# 3. Remplacement par le mode (variable categorielle)
df['ville'] = df['ville'].fillna(df['ville'].mode()[0])

# 4. Remplacement par groupe (moyenne par categorie)
df['note'] = df.groupby('classe')['note'].transform(lambda x: x.fillna(x.mean()))

# 5. Supprimer si trop de NaN dans la colonne
seuil = 0.5  # supprimer si + de 50% NaN
df = df.dropna(thresh=int(len(df) * seuil), axis=1)
Attention : Ne jamais imputer des valeurs sans raisonner sur le contexte. Un NaN peut signifier "donnee absente" (a imputer) ou "valeur zero" (a remplacer par 0) ou "non applicable" (a garder ou supprimer).

Detection et Traitement des Outliers
# Methode 1 : Z-score (valeur aberrante si |z| > 3)
from scipy import stats
z_scores = np.abs(stats.zscore(df['col']))
df_clean = df[z_scores < 3]

# Methode 2 : IQR (Interquartile Range)
Q1 = df['col'].quantile(0.25)
Q3 = df['col'].quantile(0.75)
IQR = Q3 - Q1
lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR
df_clean = df[(df['col'] >= lower) & (df['col'] <= upper)]

# Methode 3 : Percentiles
p01 = df['col'].quantile(0.01)
p99 = df['col'].quantile(0.99)
df['col'] = df['col'].clip(p01, p99)  # ecreter plutot que supprimer

# Visualiser les outliers : boxplot
import matplotlib.pyplot as plt
df.boxplot(column='col')
plt.show()
IQR (Intervalle Interquartile) : valeurs entre Q1 - 1.5*IQR et Q3 + 1.5*IQR sont considerees normales. Au-dela = outliers. C'est la methode utilisee par le boxplot de Matplotlib/Seaborn.

Frameworks de Visualisation — Comparaison Detaillee

Matplotlib

  • Bibliotheque de base, tres flexible
  • Graphiques statiques (PNG, PDF, SVG)
  • Controle pixel par pixel
  • Verbeux mais precis
  • Usage : publications, rapports PDF
  • plt.plot(), plt.bar(), plt.hist()

Seaborn

  • Construit sur Matplotlib
  • API simple, esthetique moderne
  • Specialise en statistique
  • Themes, palettes integrees
  • Usage : EDA (analyse exploratoire)
  • sns.heatmap(), sns.pairplot()

Plotly

  • Graphiques interactifs (zoom, hover)
  • Export HTML, PNG, SVG
  • Plotly Express (API simple) + Go (avance)
  • Animations, cartes geographiques
  • Usage : dashboards, web, presentations
  • px.bar(), px.scatter(), px.treemap()

Dash (by Plotly)

  • Framework web Python complet
  • Callbacks pour interactivite dynamique
  • Composants : Dropdown, Slider, Table
  • Deploiement sur serveur (Flask)
  • Usage : dashboards professionnels
  • dcc.Graph(), dcc.Dropdown()

Bokeh

  • Graphiques interactifs pour le web
  • Serveur Bokeh pour apps temps reel
  • Widgets interactifs natifs
  • Bonne performance sur gros volumes
  • Usage : apps data en temps reel

Pandas Plot

  • Directement sur DataFrame : df.plot()
  • Utilise Matplotlib en arriere-plan
  • Rapide pour exploration
  • Moins de personnalisation
  • df.plot.bar(), df.plot.hist()

Regles de Choix — Aide-Memoire

Question a se poser

  • Je compare des categories ? → Barres
  • Je montre une evolution temporelle ? → Ligne ou Aire
  • Je montre des proportions d'un tout ? → Camembert (max 6 cat.)
  • Je montre une distribution ? → Histogramme ou Boxplot
  • Je montre une relation entre 2 var. ? → Scatter plot
  • Je montre une hierarchie ? → Treemap ou Sunburst
  • Je montre une correlation ? → Heatmap
  • Je montre des outliers ? → Boxplot

Erreurs courantes a eviter

  • Camembert avec trop de categories (illisible > 6)
  • Axe Y qui ne commence pas a 0 sur barres (trompeur)
  • Ligne pour des categories non-ordonnees
  • 3D inutile (distord la perception)
  • Trop de couleurs sur un meme graphique
  • Pas de titre ni d'unites sur les axes
  • Histogramme avec trop peu de bins (perd l'info)
Dashboard Decisionnel
Definition et Objectifs Principaux

Un dashboard decisionnel est une interface visuelle qui synthetise les indicateurs cles de performance (KPIs) pour permettre une prise de decision rapide et eclairee par les managers et dirigeants.

  • Aide a la decision : informations cles pour decisions strategiques rapides
  • Vue synthetique : essentiel des KPIs en un seul ecran, pas de surcharge d'information
  • Actualisation : donnees mises a jour automatiquement (temps reel ou quasi-reel)
  • Interactivite : filtres, drilldowns, selections dynamiques pour explorer les donnees
  • Accessibilite : lisible par des non-techniciens (managers, dirigeants, clients)
  • Regle des 3 clics : toute information doit etre accessible en 3 clics maximum
  • Responsive : adapte a differentes tailles d'ecran (PC, tablette, mobile)
Composition d'un Dashboard Professionnel

Elements obligatoires

  • Titre & contexte : nom du dashboard, periode, source
  • KPIs cards : metriques cles en haut (CA, nb clients, taux...)
  • Graphiques principaux : 2-4 graphiques complementaires
  • Filtres / Slicers : par date, region, categorie, produit
  • Legendes & etiquettes : chaque graphique auto-explicatif
  • Date de mise a jour : fraicheur des donnees visible

Elements complementaires

  • Tableaux detailles : donnees brutes consultables
  • Jauges (Gauges) : progression vers un objectif cible
  • Cartes geographiques : repartition spatiale
  • Fleches de tendance : comparaison vs periode precedente
  • Alertes visuelles : codes couleurs rouge/orange/vert
  • Drilldown : exploration hierarchique (annee → mois → jour)
Utilite des Filtres dans un Dashboard
  • Permettre a l'utilisateur de se concentrer sur un sous-ensemble des donnees (region, periode, produit)
  • Rendre le dashboard interactif et personnalisable selon le besoin de chaque utilisateur
  • Eviter de creer plusieurs dashboards pour chaque cas d'usage
  • Les filtres propagent leur effet sur tous les visuels de la page simultanement
  • En Pandas : df[df['region'] == 'Nord'] | En Plotly Dash : dcc.Dropdown() | En Power BI : Slicers
Types de filtres courants : periode temporelle (date range), categorie produit, region geographique, segment client, seuil numerique (note > X), statut (actif/inactif)
Creation d'un Dashboard avec Dash (Python)
import dash
from dash import dcc, html, Input, Output
import plotly.express as px
import pandas as pd

app = dash.Dash(__name__)

# Layout = structure HTML + composants
app.layout = html.Div([
    html.H1('Dashboard Ventes 2024', style={'textAlign': 'center'}),

    # Filtres
    html.Div([
        dcc.Dropdown(id='region-filter',
                    options=[{'label': r, 'value': r} for r in df['region'].unique()],
                    value=None, placeholder='Toutes les regions', multi=True),
        dcc.DatePickerRange(id='date-range',
                            start_date=df['date'].min(),
                            end_date=df['date'].max()),
    ]),

    # KPIs
    html.Div([
        html.Div([html.H3(id='kpi-ca'), html.P('Chiffre d\'affaires')], className='kpi-card'),
        html.Div([html.H3(id='kpi-clients'), html.P('Clients')], className='kpi-card'),
    ]),

    # Graphiques
    dcc.Graph(id='bar-chart'),
    dcc.Graph(id='line-chart'),
])

# Callback = logique d'interactivite
@app.callback(
    [Output('bar-chart', 'figure'), Output('kpi-ca', 'children')],
    [Input('region-filter', 'value')]
)
def update_dashboard(regions):
    df_f = df if not regions else df[df['region'].isin(regions)]
    fig = px.bar(df_f.groupby('produit')['ventes'].sum().reset_index(),
                x='produit', y='ventes')
    kpi = f'{df_f["ventes"].sum():,.0f} MAD'
    return fig, kpi

app.run_server(debug=True)
Power BI — Business Intelligence
Qu'est-ce que Power BI ?

Power BI est un outil de Business Intelligence developpe par Microsoft permettant de connecter, transformer, modeliser et visualiser des donnees sans code approfondi. Il est massivement utilise en entreprise.

Composants principaux

  • Power BI Desktop : creation de rapports (Windows seulement)
  • Power BI Service : cloud, partage, collaboration (powerbi.com)
  • Power BI Mobile : consultation sur iOS/Android
  • Power Query : transformation des donnees (ETL, sans code)
  • DAX : langage de calcul (Data Analysis Expressions)
  • Power Pivot : modelisation des relations entre tables

Avantages vs Python

  • Interface graphique, pas de code requis
  • Connexion directe a 100+ sources de donnees
  • Actualisation automatique planifiee
  • Partage facile avec les decideurs non-tech
  • Tableaux de bord certifies et securises
  • Moins flexible pour traitements complexes

Concepts Cles Power BI
  • Slicer : filtre visuel interactif sur le rapport (equivalent dropdown Plotly)
  • Modele de donnees : relations entre tables (cle primaire ↔ cle etrangere, comme en SQL)
  • Mesure vs Colonne calculee : Mesure = calcul dynamique selon le contexte de filtre (efficace) ; Colonne calculee = stockee dans la table (moins efficace)
  • Rapport vs Dashboard : Rapport = pages detaillees editables ; Dashboard = vue synthetique epinglee (lecture seule)
  • Actualisation planifiee : mise a jour automatique depuis la source (Excel, SQL, API...)
  • RLS (Row Level Security) : restriction des donnees visibles selon le profil de l'utilisateur connecte
  • Workspace : espace de travail collaboratif dans Power BI Service
  • Gateway : passerelle pour connecter des sources locales au cloud Power BI Service
  • Schema en etoile : architecture recommandee = 1 table de faits + tables de dimensions
Schema en etoile (Star Schema) : Table de faits (transactions, ventes) liee a plusieurs tables de dimensions (produits, clients, dates, regions). C'est le modele optimal pour Power BI et les outils BI en general.
